**Extended flow (keep existing steps 1-7, replace step 8):**

Steps 1-7 remain the same (read, sanitize, build prompt, generate script, normalize speakers, validate speakers, chunk, TTS synthesis). The `audio_segments: list[bytes]` output from step 7 is the input to the new steps.

New steps after TTS synthesis:

8. **Extract episode metadata:**
   - Title: extract from first H1 heading in `content` (original markdown, before sanitization). Use regex `r"^#\s+(.+)$"` with MULTILINE. Fall back to Path(config.source_file).stem.
   - Description: first 200 characters of `clean_content` (sanitized content), stripped and truncated at word boundary.
   - Episode number: `EpisodeRepository(session).get_next_episode_number()`

9. **Process audio:**
   - Call `process_audio(audio_segments, crossfade_ms=settings.crossfade_ms, target_dbfs=settings.target_dbfs)` to get combined AudioSegment
   - Compute duration_seconds from combined.duration_seconds (pydub property)

10. **Export MP3:**
    - Build filename using sanitize_filename: `{date} - {sanitized_title}.mp3` where date is today (YYYY-MM-DD)
    - Ensure settings.episodes_dir exists (os.makedirs)
    - Export path: `{settings.episodes_dir}/{filename}`
    - Call `export_tagged_mp3(combined, export_path, title=title, artist=settings.podcast_name, track_number=episode_number)`
    - Get file_size from os.path.getsize(export_path)

11. **Persist Episode to database:**
    - Create Episode domain model with all fields
    - `EpisodeRepository(session).create(episode)`
    - `session.commit()`

12. **Regenerate RSS feed:**
    - Get all episodes from EpisodeRepository
    - Call `build_podcast_feed(settings.podcast_name, "Knowledge podcast", settings.base_url, all_episodes)`
    - Validate with `validate_podcast_rss()`. Log warnings if validation errors (do not fail pipeline).
    - Write RSS XML to `{settings.episodes_dir}/feed.xml`

13. **Write to Obsidian vault:**
    - Call `write_episode_to_vault(episode, export_path, script, settings.vault_output_dir)` where `script` is the full generated script (the transcript)
    - Log the vault output paths

14. **Return Episode** (not the file path)

**Error handling:**
- Wrap steps 9-10 in try/except for EncodingError
- Wrap step 12 in try/except for RSSError (log error, don't fail pipeline)
- If MP3 export fails, do not persist Episode or write to vault

**Update existing tests** in `tests/test_podcast_service.py`:
- The existing tests mock the TTS/LLM calls. Extend mocks to also mock the new dependencies (EpisodeRepository, audio processing, RSS, Obsidian writer).
- Add a test that verifies the new flow: given mocked TTS output bytes, the function calls process_audio, export_tagged_mp3, creates Episode, calls build_podcast_feed, calls write_episode_to_vault.
- Use unittest.mock.patch for the infrastructure calls.
