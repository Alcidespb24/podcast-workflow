  <action>
Replace the contents of `src/infrastructure/audio.py` with a pydub-based audio processing module. Keep `write_wav()` for backward compatibility (it may still be used elsewhere) but add the new pipeline functions.

**Functions to implement:**

1. `rms_normalize(seg: AudioSegment, target_dbfs: float = -20.0) -> AudioSegment`
   - If seg.rms == 0, return seg unchanged (silence)
   - Calculate gain = target_dbfs - seg.dBFS
   - Return seg.apply_gain(gain)

2. `process_audio(pcm_chunks: list[bytes], *, sample_width: int = 2, frame_rate: int = 24000, channels: int = 1, crossfade_ms: int = 30, target_dbfs: float = -20.0) -> AudioSegment`
   - Convert each PCM bytes to AudioSegment using the AudioSegment constructor (data=, sample_width=, frame_rate=, channels=)
   - RMS normalize each segment to target_dbfs
   - If only one segment, return it directly
   - Crossfade-append segments sequentially. Before each append, clamp crossfade to `min(crossfade_ms, len(current_combined), len(next_seg))` to prevent ValueError on short segments
   - Return combined AudioSegment

3. `export_tagged_mp3(audio: AudioSegment, output_path: str, *, title: str, artist: str, track_number: int) -> None`
   - Wrap in try/except to catch pydub/ffmpeg errors and raise EncodingError
   - Export: `audio.export(output_path, format="mp3", bitrate="128k", parameters=["-ac", "1"])`
   - Apply ID3 tags using mutagen EasyID3: title, artist, tracknumber (as string)
   - If ID3NoHeaderError on read, create header first with `from mutagen.id3 import ID3; ID3().save(output_path)` then open with EasyID3

**Anti-patterns to avoid:**
- Do NOT use pydub's built-in `normalize()` (it's peak-based, not RMS). Use the custom `rms_normalize`.
- Do NOT concatenate raw PCM bytes (`b"".join()`). Always use pydub AudioSegment.
- Do NOT use pydub's `tags` parameter in export(). Use mutagen for reliable ID3 tagging.

Write tests in `tests/test_audio_processing.py`. Generate test PCM data using `AudioSegment.silent(duration=1000)` or create synthetic sine waves.
