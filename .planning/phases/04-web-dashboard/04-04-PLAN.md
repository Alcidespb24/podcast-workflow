<task type="auto">
  <name>Task 2: Fix toast auto-dismiss by querying document-level toast container</name>
  <files>
    src/backend/web/static/app.js
  </files>
  <action>
**Root cause:** The `htmx:oobAfterSwap` handler uses `event.target.querySelectorAll(".toast")` but `event.target` is the main HTMX swap target (e.g. `#host-list`). The `#toast-container` is a sibling of `#main-content` in `base.html`, not a descendant of the swap target. So the query returns an empty NodeList and no `setTimeout` is ever registered. Toasts persist forever and accumulate.

**Fix — rewrite `app.js`:**

Replace the entire `htmx:oobAfterSwap` listener with one that:
1. Queries `document.querySelectorAll('#toast-container .toast')` (document-level, not event.target)
2. Skips toasts that already have a dismiss timer by checking for a `data-dismiss-scheduled` attribute
3. Marks each toast with `data-dismiss-scheduled="true"` before scheduling the timeout

The full updated app.js should be:

```javascript
/* Toast auto-dismiss and modal close handling */

document.addEventListener("htmx:oobAfterSwap", function () {
    var toasts = document.querySelectorAll("#toast-container .toast");
    toasts.forEach(function (toast) {
        if (toast.getAttribute("data-dismiss-scheduled")) {
            return;
        }
        toast.setAttribute("data-dismiss-scheduled", "true");
        setTimeout(function () {
            toast.classList.add("toast--dismiss");
            setTimeout(function () {
                toast.remove();
            }, 300);
        }, 3000);
    });
});

document.addEventListener("closeModal", function () {
    var dialogs = document.querySelectorAll("dialog[open]");
    dialogs.forEach(function (dialog) {
        dialog.close();
        dialog.remove();
    });
});
```

Key changes from original:
- `event.target.querySelectorAll(".toast")` becomes `document.querySelectorAll("#toast-container .toast")`
- Remove the unused `event` parameter from the callback
- Add `data-dismiss-scheduled` guard to prevent double-scheduling if multiple OOB swaps fire
- The `closeModal` handler is unchanged
